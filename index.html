Quick-Start Guide

How to submit and manage data entries

1
Open the relevant form
Click any card above. Forms load the latest saved submissions automatically from GitHub — no login required.
2
Fill in the fields
Fields marked * are required. Some fields (e.g. Date Of Call For RPS, RPS Amount) are auto-computed — they fill in automatically once you enter the source field.
3
Save the submission
Click + Save Submission. The data is committed directly to the GitHub repository and immediately visible to all users who refresh their page. A confirmation alert will appear once saved.
4
Start a new entry
Click New Submission to clear the form and begin a fresh entry. Your previous submission remains saved.
5
Edit a saved entry
In the Saved Submissions table at the bottom of the form, click Edit next to any row. The form re-populates with that entry's data. Make your changes, then click ✓ Update Submission to save.
6
Delete an entry
Click Delete next to any saved submission. You will be asked to confirm. Deletion is permanent and committed to GitHub.
7
Download the Word report
Click ⬇ Download Word to export all saved submissions for that form into a single .docx file. Each submission appears as a labelled block with an edit link. The file is always named the same (e.g. Disbursement-DataSheet.docx) so it overwrites the previous version when saved to the same folder.
8
Refresh to see other users' changes
Submissions are loaded once on page open. Click ↻ Refresh in the Saved Submissions panel header to pull the latest data from GitHub without reloading the page.
ℹ️
Supported browsers: Google Chrome and Microsoft Edge. Other browsers may not support all features (Word export requires a modern browser).
⚠️
After saving, Netlify redeploys automatically. Allow ~30 seconds before other users will see new submissions on a fresh page load. Use ↻ Refresh to get changes immediately without waiting for a redeploy.
